# Contributing to QuantumVault

## Security Considerations

QuantumVault is a security-focused project. When contributing, please:

1. **Security First**: All changes must prioritize security
2. **Code Review**: All PRs require security review
3. **Testing**: Include comprehensive security tests
4. **Documentation**: Update security documentation

## Development Guidelines

### Setting Up Development Environment

1. Linux environment for encryption component
2. DJGPP for DOS component compilation
3. DOSBox or similar for testing

### Code Style

- Clear, documented code
- Comprehensive error handling
- Security-focused logging
- Memory-safe practices

### Testing Requirements

- Unit tests for all components
- Integration tests
- Security validation tests
- Cross-platform testing

## Pull Request Process

1. Create feature branch
2. Add tests and documentation
3. Update security documentation
4. Submit for review

## Security Disclosure

For security issues, please email: security@harrisonsec.com
